# Quick Start

Get started with Kumo by building a simple example. This guide walks you through creating your first component.

<Steps>

<Step title="Install Kumo">

If you haven't already, install Kumo and its dependencies:

<CodeGroup>

```bash npm
npm install @cloudflare/kumo react react-dom @phosphor-icons/react
```

```bash yarn
yarn add @cloudflare/kumo react react-dom @phosphor-icons/react
```

```bash pnpm
pnpm add @cloudflare/kumo react react-dom @phosphor-icons/react
```

</CodeGroup>

Then import styles in your main CSS or JS file:

```js
import "@cloudflare/kumo/styles/standalone";
```

</Step>

<Step title="Import components">

Import components from Kumo. You can use the main package export or granular imports:

```tsx
// Main package import
import { Button, Badge } from "@cloudflare/kumo";

// Granular imports (recommended for tree-shaking)
import { Button } from "@cloudflare/kumo/components/button";
import { Badge } from "@cloudflare/kumo/components/badge";
```

</Step>

<Step title="Use your first component">

Create a simple component using Button and Badge:

```tsx Example.tsx
import { Button, Badge } from "@cloudflare/kumo";
import { Plus } from "@phosphor-icons/react";

function Example() {
  return (
    <div style={{ padding: "2rem" }}>
      <div style={{ display: "flex", gap: "1rem", alignItems: "center" }}>
        <Button variant="primary" icon={Plus}>
          Create Worker
        </Button>
        <Button variant="secondary">
          View All
        </Button>
        <Badge variant="success">Active</Badge>
      </div>
    </div>
  );
}

export default Example;
```

</Step>

</Steps>

## Working Example

Here's a more complete example showing different component variants:

```tsx App.tsx
import { Button, Badge, Surface } from "@cloudflare/kumo";
import { Rocket, Eye, TrashSimple } from "@phosphor-icons/react";

function App() {
  return (
    <Surface className="p-6 max-w-2xl">
      <div className="space-y-4">
        <div>
          <h1 className="text-2xl font-bold mb-2">Welcome to Kumo</h1>
          <Badge variant="beta">Beta</Badge>
        </div>
        
        <div className="flex gap-2">
          <Button variant="primary" icon={Rocket}>
            Deploy
          </Button>
          <Button variant="secondary" icon={Eye}>
            Preview
          </Button>
          <Button variant="destructive" icon={TrashSimple}>
            Delete
          </Button>
        </div>

        <div className="flex gap-2">
          <Badge variant="success">Active</Badge>
          <Badge variant="primary">Production</Badge>
          <Badge variant="secondary">v1.9.0</Badge>
        </div>
      </div>
    </Surface>
  );
}

export default App;
```

## Component Variants

Most Kumo components support multiple variants. Here are the Button variants:

```tsx
// Primary action (high emphasis)
<Button variant="primary">Save Changes</Button>

// Secondary action (default)
<Button variant="secondary">Cancel</Button>

// Minimal button
<Button variant="ghost">Learn More</Button>

// Destructive action
<Button variant="destructive">Delete</Button>

// Icon-only button (requires aria-label)
<Button variant="secondary" shape="square" icon={Plus} aria-label="Add item" />
```

## Using Icons

Kumo uses icons from [@phosphor-icons/react](https://phosphoricons.com). Import and pass them to the `icon` prop:

```tsx
import { Button } from "@cloudflare/kumo";
import { CloudArrowUp, Database, GearSix } from "@phosphor-icons/react";

function IconExamples() {
  return (
    <>
      <Button icon={CloudArrowUp}>Upload</Button>
      <Button icon={Database}>View Data</Button>
      <Button icon={GearSix} shape="square" aria-label="Settings" />
    </>
  );
}
```

## Loading States

Many components support loading states:

```tsx
<Button variant="primary" loading>
  Saving...
</Button>
```

## Explore the CLI

Kumo includes a built-in CLI for quick component reference:

```bash
# List all components
npx @cloudflare/kumo ls

# Get component documentation
npx @cloudflare/kumo doc Button

# Browse installable blocks
npx @cloudflare/kumo blocks
```
