## Description

Password/secret input that masks its value by default and reveals on click. Includes a built-in copy-to-clipboard button on hover.

Perfect for API keys, tokens, passwords, and other sensitive data that needs to be obscured but occasionally revealed or copied.

## Import

```tsx
import { SensitiveInput } from "@cloudflare/kumo";
```

## Features

### Masking

- Value is masked by default with bullet characters (`•••••••`)
- Click the eye icon to toggle visibility
- Masked state label shows when label prop is provided

### Copy to Clipboard

- Hover over the input to reveal the copy button
- Click to copy the value to clipboard
- Visual feedback when copied

### Security

- Input uses `type="password"` when masked for browser security
- Switches to `type="text"` when revealed
- No value leaking in browser autocomplete

## Styling

SensitiveInput uses semantic Kumo tokens:

- **Background**: `bg-kumo-control`
- **Text**: `text-kumo-default`
- **Placeholder**: `text-kumo-subtle`
- **Button**: `bg-kumo-brand` (copy button)
- **Ring**: `outline-kumo-ring`

## Accessibility

- Proper label association via Field wrapper
- Aria labels for icon buttons (reveal/hide, copy)
- Keyboard accessible
- Screen reader friendly

## When to Use

### Use SensitiveInput for:
- API keys and tokens
- Passwords (in settings/profile pages, not login forms)
- Secret keys and credentials
- Any value that should be obscured but needs occasional reveal/copy

### Use regular Input with `type="password"` for:
- Login forms
- Password creation/confirmation forms
- Any password field that shouldn't have a reveal button
