<Note>
This component includes both `CloudflareLogo` (logo mark) and `PoweredByCloudflare` (badge component).
</Note>

## Import

```tsx

```

```tsx

```

## CloudflareLogo Props

<ParamField path="variant" type="'glyph' | 'full'" default="full">
Logo variant:
- `glyph` — Cloud icon only (logomark)
- `full` — Full logo with cloud glyph and wordmark
</ParamField>

<ParamField path="color" type="'color' | 'black' | 'white'" default="color">
Color scheme:
- `color` — Brand colors (orange/yellow cloud, dark gray wordmark)
- `black` — Solid black
- `white` — Solid white (for dark backgrounds)
</ParamField>

<ParamField path="className" type="string">
Additional CSS classes merged via `cn()`.
</ParamField>

<ParamField path="...props" type="SVGAttributes<SVGSVGElement>">
All standard SVG element attributes (width, height, aria-label, etc.).
</ParamField>

## Variants

### CloudflareLogo Variants

#### Glyph

```tsx
<CloudflareLogo variant="glyph" className="w-12" />
```

- ViewBox: `0 0 49 22`
- Contains: Cloud icon only (orange and yellow gradient)

#### Full (default)

```tsx
<CloudflareLogo variant="full" className="w-40" />
```

- ViewBox: `0 0 101.4 33.5`
- Contains: Cloud icon + "CLOUDFLARE" wordmark

### Color Schemes

#### Color (default)

```tsx
<CloudflareLogo color="color" />
```

- Cloud: Orange (`#F48120`) and yellow (`#FAAD3F`) gradient
- Wordmark: Dark gray (`#404041`) in color mode, respects theme in dark mode via `text-kumo-default`

#### Black

```tsx
<CloudflareLogo color="black" />
```

- All elements: `currentColor` with `text-black` class

#### White

```tsx
<CloudflareLogo color="white" />
```

- All elements: `currentColor` with `text-white` class

## SVG Generation Helper

Generate SVG markup string for copying to clipboard or embedding:

```tsx
import { generateCloudflareLogoSvg } from "@cloudflare/kumo";

// Copy glyph SVG to clipboard
const svg = generateCloudflareLogoSvg({ variant: "glyph" });
await navigator.clipboard.writeText(svg);

// Generate full logo in black
const blackSvg = generateCloudflareLogoSvg({ variant: "full", color: "black" });
```

### Options

```tsx
interface GenerateCloudflareLogoSvgOptions {
  variant?: "glyph" | "full"; // default: "full"
  color?: "color" | "black" | "white"; // default: "color"
}
```

## Accessibility

- Both components use semantic SVG with `role="img"`
- Includes `aria-label="Cloudflare logo"`
- PoweredByCloudflare is a link with `target="_blank" rel="noopener noreferrer"`
- All components support standard ARIA attributes

## Brand Guidelines

- Use `color` variant on light backgrounds for maximum brand recognition
- Use `white` variant on dark backgrounds
- Use `black` variant when color printing is not available
- Maintain aspect ratio by setting only `width` or `height` with `className`
- Ensure adequate padding around the logo (minimum 0.5x logo height)
- For full brand guidelines, visit [cloudflare.com/brand-assets/](https://www.cloudflare.com/brand-assets/)

## Notes

- Logo colors are hard-coded brand colors when `color="color"`
- In `color` mode, wordmark uses `text-kumo-default` to respect dark mode
- SVG uses `currentColor` for `black` and `white` variants
- PoweredByCloudflare always opens link in new tab
- Component uses `forwardRef` for ref forwarding
